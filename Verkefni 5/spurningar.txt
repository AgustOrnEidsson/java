1. Vegna þess að það leitar í öllu DOM tréinu. getElementById() hættir þegar það finnur id, það gerir það vegna þess að það á ekki að vera annað alvegeins id.                                                

2. querySelectorAll() er mun dýrari en getElementById(). Það velur allt sem hefur sama CSS identifier og setur það allt í lista.

3. var elements = document.getElementsByTagName('li');
   if (elements.length > 0) {  //Nodelist hefur length property//
 	var el = elements[2];  //velur þriðja element//
 	el.className = 'cool'; //hægari aðferð sem NodeList á, breytir classanum í cool//
        }


4. Þetta velur öll elements sem eru með tagið sem þú hefur valið t.d ef þú velur tagið p þá skilar það öllu p-i í lista.
	Dæmi:
		var elementid = document.getElementsByTagName('p');
		"allt p"

5. Flest allir browserar bæta við next node þar sem er whitespace á milli elements. Auðu bilin fara í DOM tréin og geta búið til vandræði. 
   Þú getur leist þetta vandamál með því að færa aftasta gogginn að næsta elementi
   Dæmi:
	 <h1>Halló Heimur!</h1
         ><p>Þetta er fyrsta vefsíðan mín</p>
